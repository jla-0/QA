## Resume Honesty Guardrails

The job asks for claims processing, SOAP UI, EDI, Oracle, SQL Server, automated testing, and performance testing. The resume should not imply paid experience in every one of those areas if that was not the case.

The tailored approach separates experience into three categories:

| Category | Resume treatment |
|---|---|
| Direct paid experience | Regulated systems QA, validation testing, test evidence, traceability, SQL-backed validation, healthcare IT, HIPAA-aware workflows, defect/deviation documentation, cross-functional QA communication |
| Adjacent technical capability | SQL, data analysis, backend validation, integration documentation, web/API concepts, data integrity, audit trails, access controls |
| Portfolio proof | Healthcare claims workflow, EDI sample files, SOAP/XML request and response checks, Jira-style defects, load testing plan, claims-specific test cases |

## Suggested Application Positioning

Use this framing in an application or recruiter message:

> Regulated QA background from pharmaceutical validation and healthcare IT, with hands-on experience creating test evidence, traceability, SQL-backed validation, defect documentation, and audit-ready quality records. A claims-focused QA portfolio project shows coverage for claim intake, adjudication, EDI/SOAP integrations, SQL backend behavior, HIPAA-safe data, Jira defects, regression, and release readiness.

## Interview Risk And Response

If asked whether the professional background includes direct medical claims processing, answer plainly:

> Direct healthcare background includes clinical IT systems, PACS/DICOM workflows, HIPAA-aware support, and regulated system documentation. Direct QA background includes validated systems, SQL/data checks, requirements traceability, test execution, and defect-style investigations. Because this role is claims-specific, the portfolio walkthrough uses synthetic claims data, EDI examples, SOAP/XML checks, SQL validation, and Jira-style defects to show a practical claims QA approach.

That answer is strong because it is honest, prepared, and evidence-backed.
