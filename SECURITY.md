# Security And Data Handling

This repository is designed for public portfolio use. It must not contain PHI, credentials, access tokens, real claim data, real member data, real provider data, or employer-confidential information.

## Reporting A Data Concern

If sensitive data is ever found in this repository, remove it from the current branch immediately and rotate any exposed credentials if applicable.

## Data Classification

| Data type | Allowed |
|---|---|
| Synthetic claim IDs | Yes |
| Synthetic member IDs | Yes |
| Synthetic provider IDs | Yes |
| Real patient or member information | No |
| Real claim data | No |
| Credentials or tokens | No |
| Employer-confidential workflows | No |
